# 三维城市规划协同管理系统 - 全栈开发记录 (Agent.md)

## 一、项目概述与功能列表

本项目旨在构建一个基于 Vue3 + Cesium 和 Spring Boot 的“三维城市规划协同管理系统”。系统通过三维地球进行项目宏观与微观的展示，并提供基于数字字典的强业务关联与附件管理，支持高颗粒度的角色权限控制（管理员与普通用户的操作差异）。

### 后端核心功能列表

1. **基础业务字典管理 (Dict)**：支持系统全局各类下拉选项的统一配置（大类、子类字典）。
2. **空间项目数据管理 (Project)**：支持 `ST_GeomFromGeoJSON` 空间特性数据的入库和基于业务维度（项目类别、进度、地区）的分页/组合查询及项目各项指标统计。
3. **安全附件管理 (Project File)**：基于业务项目的关联附件上传、列表读取与流式文件下载功能。
4. **天地图代理服务 (Map Proxy)**：解决前端 Cesium 加载天地图服务时的 CORS 跨域拦截方案。
5. **系统启动数据初始化**：开箱即用的 GeoJSON 底图数据与默认字典的预载及自修补能力。

### 前端体验功能列表

1. **3D 引擎展示**：Cesium 驱动的立体地球渲染、工具视角控制配置及全行政区划 GeoJSON 可视化加载（含 MultiPolygon 支持）。
2. **模块解耦式渲染**：首页专注“六大片区”三维展板，其他页面专注具体业务细分（土地储备、工程项目等）。
3. **定制化交互浮层**：完全摒弃原生 Cesium Popup，纯 CSS 开发的高级极客科幻风半透明弹层看板，具备动态图表、彩色渐变与管理动作。
4. **权限感知 UI**：普通人员只读，管理人员可以在面板中点击直接唤起进入“片区/项目信息修改”表单页面。

## 三、数据库实体关系与表结构说明

系统使用 **PostgreSQL** 作为核心关系型数据库，并深度依赖 **PostGIS** 扩展插件来存储和执行 3D 地理空间数据处理（如 `ST_GeomFromGeoJSON` 空间类型转换）。

### 主要数据表一览表

| 表名 (Table) | 描述 (Description) | 核心字段说明 |
| --- | --- | --- |
| `sys_user` | **系统用户表** | 区分 `ADMIN`、 `SUPER_ADMIN` 和 `USER` 角色。涉及鉴权与数据隔离。 |
| `biz_project` | **项目主数据表** | 存储所有的片区、工程、土地项目的属性。包括关联的 `geometry` (PostGIS 面状空间字段) 和 `extra_data` (用于存储灵活的附属数据，如投资金额等 JSON 数据)。 |
| `biz_project_file` | **项目附件关联表** | 记录通过文件上传接口录入的图纸、文档元信息及其关联的项目 ID。 |
| `sys_dict` | **全局字典大类表** | 存储诸如 `project_category`（项目大类）等根数据。 |
| `sys_dict_item` | **全局字典细项表** | 例如存储 `PIANQUGUIHUA`（片区规划）、`COMMERCIAL`（商业用地）等带排序功能的细分字典选项。 |
| `sys_log` | **系统操作日志表** | 审计与追踪关键操作记录及接口访问踪迹。 |
| `biz_land_reserve` | **土地储备专门表** | 早期遗留表或特定复杂土地字段附属扩展用表。 |
| `biz_property` | **房产信息表** | 记录房产特定信息（编号、面积、用途、状态等）。 |
| `biz_danger_point` | **危险作业监测点表**| 记录诸如监测点类型、安全等级的空间信息表。 |
| `map_drawing` | **地图标绘表** | 存储用户在前端地图工具中手绘的点/线/面要素数据（带 GeoJSON 属性）。 |

## 四、最新模块开发与攻坚摘要

### 🟢 核心攻坚一：Cesium 地图引擎数据穿透

- 修复了加载官方天地图产生的 500 跨域/请求响应失败的问题，通过在 `DataInitializer` 中增加本地代理支持，并成功挂载带图标配置的层级底图。
- 完整重写了之前有缺陷的空间渲染器：支持了含有漏洞环 (Holes) 和 `MultiPolygon` 多边形数组格式的复杂官方地理边界数据。给行政边界设计了红色浅透高亮和粗边框效果。

### 🟢 核心攻坚二：模块级首页“六大片区”看板

- 依托于 Vue Router 的 ActiveModule 状态栏实现三维场景的**区域隔离**：让六大片区的面斑仅显示于【首页】且加载极富感染力的高亮彩边；使得普通的招拍挂项目等退居【土地储备】等专页，不干扰首屏。

### 🟢 核心攻坚三：字典关联一致性自动化自愈机制

- 原有系统在手工新建大屏区域时缺失与之绑定的字典数据，导致页面查不回“片区规划”字段类型及产生管理面板断层。
- 后端开发了 `DataInitializer.java` 作为兜底恢复探针进程，能在服务器首次启动前进行健康扫描：若系统丢失 `engineering_sub_type` / `PIANQUGUIHUA` 映射或已有 `DISTRICT` 数据的汉化错乱，该类会在不干扰用户的情况下自动在库表执行脏数据修复 `UPDATE` 和 `INSERT`。
