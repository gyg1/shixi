1、研究背景及现状。

随着我国城市化进程快速推进，城市建设规模、开发强度不断加大，城市规划管理面临着信息量大、更新频率高、协同参与主体多等新挑战。目前我国智慧城市数据平台正从“建设期”向“价值释放期”过渡，未来将进一步深化数据共享与开放，加速数据要素市场化，推动行业应用和区域协同发展。传统二维 GIS 规划手段在空间要素表达能力上存在抽象、信息损失严重等问题，已难以满足当前智慧城市建设对数据精细化、业务协同化和审批透明化的需求。

三维 GIS 能够真实表达城市空间结构，多视角立体展示规划成果，支持可视化空间分析，正在成为城市规划管理的重要技术方向。现阶段 QGIS、Cesium、SuperMap 等 GIS 软件及 WebGL 技术发展成熟，为三维城市规划应用系统建设提供了实现条件。然而当前许多系统仍仅停留在数据浏览层面，对规划业务流程融合不足，缺乏规划方案管理、多维协同决策与指标对比等深层功能。因此，有必要构建一个面向规划业务的三维城市规划协同管理平台，实现基于 Web 的多端可视化管理与决策支持，推动规划数据价值转化与业务提效。

2. 设计内容

土地储备管理：实现土地资源的电子化与空间可视化展示，实时监控其收储、在库、出让等状态变化，并支持储备规模与价值的综合分析。。

项目建设管理：对项目的立项、施工到竣工全周期进行动态跟踪与管理，支持关键阶段性成果的上报、审核与进度信息的可视化展示。

房地产销售管理：针对房地产行业重视现金流的特点，本模块不仅记录销售业绩，更重点分析销售周期与资金回笼情况，旨在帮助管理人员更全面地掌握项目运行状态。

资金与效益分析：实现对项目投资、成本、收益等关键财务数据的全面统计与核算，通过回报率等指标进行多维度经济效益评估与可视化。

危险作业监管：结合实时数据，对高风险作业进行智能监控与风险可视化。

项目信息管理：负责所有项目基础信息的录入、修改、删除与归档，为上层各类可视化分析与展示提供可靠数据支撑。

用户管理模块：实现系统用户的账号注册、身份认证、分级登录与权限控制，保障系统安全，并支持个人基本信息的维护与更新功能。

3. 设计方案及预期达到的目标。

一、技术选型

后端开发：采用 Java 语言，基于 Spring Boot 框架进行系统后端的开发。Spring Boot 具备结构清晰、开发便捷以及良好扩展性等优势，能够有效支持城乡规划多业务协同管理需求。

前端开发：使用 Vue.js 框架实现业务页面构建，并集成 CesiumJS 进行三维可视化展示，同时借助 ECharts 进行数据可视化分析，以提升用户交互体验。

数据库：选择 PostgreSQL 数据库，并结合 PostGIS 处理空间地理信息数据，实现城市地块、建筑对象等数据的精准存储、查询和分析。

GIS 支撑：利用 QGIS 对空间数据进行建模、格式转换和管理，保障三维空间数据的准确性与可用性。

二、系统架构设计

后端采用 Spring Boot 的分层架构设计，主要包括控制层、业务逻辑层和数据访问层。控制层负责处理前端请求与响应，业务逻辑层负责规划业务管理与分析计算，数据访问层负责与数据库进行交互。

前端基于 Vue.js 进行组件化开发，将不同业务模块拆分为独立组件，提高系统页面的可维护性与复用性。同时集成 CesiumJS，用于三维城市场景加载与交互操作，实现规划信息与模型数据的立体化表达。

数据层由 PostgreSQL + PostGIS 构成，既能管理普通业务数据，也支持空间数据存储与空间分析算法调用，为土地储备、项目建设、房地产销售等业务提供可靠数据支撑。

三、数据库设计

根据系统五大核心功能模块，设计对应的数据表结构，包括土地储备信息表、项目建设进度表、房地产销售数据表、资金与效益分析指标表以及危险作业监测数据表等。数据库中不仅存储对象属性数据，同时利用 PostGIS 建立空间字段，用于保存地块边界、楼栋位置和监测点位等空间信息。通过合理规划数据表关系和索引，可确保系统的数据存储性能和检索效率，并保证信息安全性与完整性。

4、进度计划。

第一阶段：需求分析与系统设计阶段，进行相关资料查阅与调研，明确系统业务流程与功能需求；完成系统总体架构设计、数据库表结构设计以及前后端接口规范制定；输出高保真原型图，撰写并完善开题报告。

第二阶段：核心功能开发与实现阶段，搭建前后端开发环境，完成基础框架构建；集成Cesium三维引擎与ECharts图表库；依次完成土地储备、项目建设、房地产销售及资金效益分析等核心业务模块的代码编写与功能实现；完成各模块间的数据交互与联调。
