# 接口清单文档

## 1. 文档说明

本文档基于当前后端控制器代码整理，描述系统现有接口清单，用于后续前后端联调、迭代开发和接口治理。

后端服务默认地址：

- `http://localhost:8080`

接口统一前缀：

- `/api`

统一响应结构：

```json
{
  "code": 200,
  "message": "success",
  "data": {},
  "timestamp": 1710000000000
}
```

说明：

- 大部分接口返回 `ApiResponse<T>`
- 少数文件下载接口直接返回文件流
- 少数地图代理接口直接返回第三方原始 JSON 字符串

## 2. 权限约定

### 2.1 角色

- `USER`
- `ADMIN`
- `SUPER_ADMIN`

### 2.2 权限规则概览

- 公开接口
  - `/api/auth/**`
  - `/api/public/**`
  - `/api/projects/files/*/download`
- 超级管理员专用
  - `/api/users/**`
- 管理员及以上可写
  - 项目写接口
  - 字典写接口
  - 表单模板写接口
  - 文件上传/删除
  - 土地储备写接口
  - 房产写接口
- 已登录用户可访问
  - 多数业务查询接口
  - 地图标绘接口

## 3. 认证接口

控制器：

- `AuthController`

基础路径：

- `/api/auth`

### 3.1 登录

- 方法：`POST`
- 路径：`/api/auth/login`
- 权限：公开
- 请求体：

```json
{
  "username": "admin",
  "password": "admin123"
}
```

- 说明：
  - 用户登录
  - 返回 JWT、过期时间和用户信息

### 3.2 注册

- 方法：`POST`
- 路径：`/api/auth/register`
- 权限：公开
- 请求体：

```json
{
  "username": "test",
  "password": "123456",
  "email": "test@example.com",
  "phone": "13800000000",
  "realName": "测试用户"
}
```

- 说明：
  - 注册普通用户

### 3.3 获取当前用户

- 方法：`GET`
- 路径：`/api/auth/me`
- 权限：已登录
- 说明：
  - 返回当前登录用户信息

### 3.4 校验 Token

- 方法：`POST`
- 路径：`/api/auth/validate`
- 权限：已登录
- 请求头：
  - `Authorization: Bearer <token>`
- 说明：
  - 校验当前 token 是否有效

### 3.5 刷新 Token

- 方法：`POST`
- 路径：`/api/auth/refresh`
- 权限：公开
- 说明：
  - 当前仅占位，未真正实现刷新逻辑

## 16. 当前接口现状结论

当前接口体系已经具备以下特点：

- 已形成完整的前后端业务基础接口
- 项目管理接口最完整
- 字典与模板接口已支撑动态表单
- 危大作业和地图标绘接口已形成独立模块
- 地图代理接口已支撑搜索和行政区边界获取
