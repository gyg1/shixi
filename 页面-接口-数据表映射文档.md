# 页面-接口-数据表映射文档

## 1. 文档说明

本文档用于建立以下三者之间的对应关系：

- 前端页面
- 后端接口
- 数据库表

目标：

- 方便后续迭代时快速定位改动范围
- 便于确认某个页面背后的接口来源与数据落表位置
- 便于发现“页面已存在但后端未补齐”或“接口已存在但页面未接入”的情况

本文档基于当前代码现状整理，属于当前基线认知，不代表所有页面都已完全联调完成。

## 3. 页面级详细映射

## 3.1 登录注册页

页面：

- `CeuimDataScreen/src/views/Login.vue`

用途：

- 用户登录
- 普通用户注册

对应接口：

- `POST /api/auth/login`
- `POST /api/auth/register`
- `GET /api/auth/me`
- `POST /api/auth/validate`

对应数据表：

- `sys_user`

读写关系：

- 登录时读取 `sys_user`
- 注册时写入 `sys_user`
- 登录后通过 `/me` 回读当前用户信息

补充说明：

- 页面本身不直接管理角色
- 注册默认创建普通用户

- `GET /api/projects/category/{category}/stats`

### 房产相关

- `GET /api/properties`
- `GET /api/properties/{id}`
- `GET /api/properties/type/{type}`
- `GET /api/properties/statistics`

### 危大作业相关

- `GET /api/dangerous-works`
- `GET /api/dangerous-works/{id}`
- `GET /api/dangerous-works/monitors`
- `GET /api/dangerous-works/statistics`
- `POST /api/dangerous-works`
- `PUT /api/dangerous-works/{id}/status`

### 地图标绘相关

- `GET /api/map-drawings`

- `biz_property`

主要读写逻辑：

- 读取项目、土地、房产、危大作业并上图
- 读取项目附件用于弹窗或详情展示
- 读写 `map_drawing` 作为用户标绘结果
- 使用地图代理接口完成地名搜索和行政区边界定位

特别说明：

- 这是当前系统最重的聚合页
- 一个页面背后串联了最多的接口和表

## 3.7 项目表单页

页面：

- `CeuimDataScreen/src/views/project/ProjectForm.vue`

用途：

- 新建项目
- 编辑项目
- 基于分类和模板动态渲染扩展字段
- 地图上绘制项目区划 geometry

对应接口：

### 项目相关

- `GET /api/projects/{id}`
- `POST /api/projects`
- `PUT /api/projects/{id}`

### 字典相关

- `GET /api/dicts/code/project_category/items`
- `GET /api/dicts/code/land_sub_type/items`
- `GET /api/dicts/code/engineering_sub_type/items`

### 模板相关

- `GET /api/form-templates/match`

### 地图相关

- `GET /api/map/search`
- `GET /api/map/administrative`

对应数据表：

- `biz_project`
- `sys_dict`
- `sys_dict_item`
- `sys_form_template`

读写关系：

- 保存项目基础字段到 `biz_project`
- 扩展字段序列化后写入 `biz_project.extra_data`
- 项目分类和子类来自字典表
- 模板配置来自 `sys_form_template`
- 地图代理接口只做辅助定位，不直接落表

重点关系：

- 页面选择 `category`
  -> 查询字典项
- 页面选择 `subType`
  -> 查询模板
- 页面填写扩展字段
  -> 写入 `extra_data`
- 页面绘制范围
  -> 写入 `geometry`

## 6. 当前映射中的注意点

### 6.1 页面聚合度不均衡

当前最重的页面是 `UserMapView.vue`，它同时连接了多个业务表和接口；而某些后台页面虽然入口存在，但业务深度尚未完全展开。

### 6.2 土地、房产更偏“地图模块”而非“后台主表页”

后端已有成熟 CRUD 接口，但前端当前更偏地图展示，独立后台管理页未像项目管理一样成体系。

### 6.3 配置驱动链条已经形成

`字典管理 -> 模板管理 -> 项目表单 -> 项目主表` 这条链已经成立，是当前系统很重要的架构特色。

### 6.4 系统设置页与持久化表尚未闭环

页面已存在，但后端没有清晰对应配置表，说明该模块目前更偏半成品或占位设计。

## 7. 当前基线结论

从页面、接口、数据表三层来看，当前系统已经形成以下主线闭环：

1. 登录认证闭环
   - 页面：登录页
   - 接口：认证接口
   - 表：`sys_user`

2. 项目管理闭环
   - 页面：项目列表、项目表单、地图项目展示
   - 接口：项目、附件、字典、模板、地图代理接口
